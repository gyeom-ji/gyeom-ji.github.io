## 운영체제 OS(Operating System)

---

- 운영체제는 컴퓨터의 주기억장치에 상주하여, 사용자가 컴퓨터의 하드웨어를 쉽게 사용할 수 있도록 인터페이스를 제공해 주는 소프트웨어이다.
- 한정된 시스템 <span style="color:#9fb584">**자원을 효과적으로 사용할 수 있도록 관리 및 운영**</span>함으로써 사용자에게 편리성을 제공한다.
- 컴퓨터 시스템과 사용자 간 <span style="color:#9fb584">**인터페이스 기능을 담당**</span>한다.
- <span style="color:#9fb584">**신뢰도(Reliability)와 처리량(Throughput)을 향상**</span>시킨다.
- <span style="color:#9fb584">**응답 시간(Response Time)을 단축**</span>한다.


![operatingSystem](/assets/img/operatingSystem.png)

<br/>

## 운영체제 역할

---

#### 1. 사용자 편리성 제공 

: 한정된 <span style="color:#9fb584">**시스템 자원을 효과적으로 사용할 수 있도록 관리 및 운영**</span>한다.

#### 2. 인터페이스 기능 담당 

: 사용자와 컴퓨터 하드웨어 간 상호작용을 위한 <span style="color:#9fb584">**인터페이스를 제공**</span>한다.

#### 3. 자원관리 

: <span style="color:#9fb584">**CPU, 메모리 공간, 기억장치, 입출력 장치 등의 자원을 관리**</span>한다.

- <span style="color:#9fb584">**CPU 스케줄링**</span> : 여러 프로세스 간 CPU를 공유하고 실행 순서를 결정한다.
- <span style="color:#9fb584">**메모리 관리**</span> : 물리적인 메모리를 가상 메모리로 추상화하고, 프로세스에게 메모리 공간을 할당, 해제한다.
- <span style="color:#9fb584">**파일 관리**</span> : 기억장소를 할당하고, 빈공간을 관리하며, 디스크 스케쥴링을 한다.
- <span style="color:#9fb584">**디바이스 관리**</span> : 입출력 장치와 통신을 조정하고 디바이스 드라이버를 관리하여 하드웨어 자원을 효율적으로 사용한다.

> 프로세스 Process : CPU에 의해 처리되는 실행 중인 프로그램

#### 4. 제어 기능 

: <span style="color:#9fb584">**입출력 장치와 사용자 프로그램을 제어**</span>한다.

- 컴퓨터 시스템의 동작을 제어하고 조정한다. 이를 통해 프로그램이 자원을 오작동 하거나 다른 프로그램에 영향을 주지 않도록 보장한다.
- 사용자와 하드웨어 간 상호작용을 관리하여 안정성과 신뢰성을 유지한다.

#### 5. 오류처리와 보안 기능

: <span style="color:#9fb584">**시스템 오류를 감지하고 처리**</span>하여 시스템의 안정성을 유지한다.

- 사용자 및 시스템 자원에 대한 접근을 제어하여 보안을 강화한다.
- 비인가된 엑세스나 악의적인 소프트웨어 실행을 방지하고, 데이터의 무결성과 기밀성을 보호한다.

## 운영체제 구성요소

---

![software](/assets/img/software.png)

### 1. 커널 Kernel

- 운영체제의 <span style="color:#9fb584">**핵심이 되는 기능들이 모여 있는 컴퓨터 프로그램으로, 기본적인 필수 기능을 수행**</span>한다.
- 컴퓨터의 <span style="color:#9fb584">**하드웨어와 추상화 자원을 관리**</span>한다. 
- 커널은 항상 컴퓨터 자원만 바라보고 있기 때문에 사용자와의 상호작용을 지원하지 않는다. (-> 사용자는 쉘을 통해 커널에게 명령을 내릴 수 있다.)
- 커널은 컴퓨터가 부팅될 때 주기억 장치에 적재된 후 상주하면서 실행하며, 프로그램과 하드웨어 간의 인터페이스 역할을 담당한다.

> 추상화 : 하나뿐인 하드웨어를 마치 여러개인 것처럼 보여지도록 하는 기술<br/> 
> 커널은 이를 위해 하나의 하드웨어 자원을 여러 사용자들을 위한 여러개의 추상화된 객체로 관리하게 된다.<br/> 
> <br/> 
> 물리적 자원 이름 -> 추상화한 자원 용어 <br/> 
> CPU -> 태스크(Task)<br/> 
> 메모리(memory) -> 페이지(page), 세그먼트(segment)<br/> 
> 디스크(disk) -> 파일(file)<br/> 
> 네트워크(network) -> 소켓(socket)

<br/> 

#### 커널의 기능

1. 프로세스 관리 
    - 프로세스 스케줄링 및 동기화 관리 담당
    - 프로세스 생성과 제거, 시작과 정지, 메시지 전달 등의 기능 담당
2. 메모리 관리
    - 프로세스에게 메모리 할당 및 회수 관리 담당
    - 물리적 메모리 관리뿐만 아니라 가상 메모리를 만들고 관리한다.
    - 커널은 물리적 메모리 또는 스왑 공간에 메모리의 각 페이지를 배치한다.
    - 커널은 어떤 페이지가 어디 있고 어디로 옮겨진지를 가리키는 메모리 페이지 테이블을 가지고 있다.
    - 메모리 페이지 사용여부를 지속적으로 추적하여, 사용할 수 있는 공간이 남아있어도 일정 시간 쓰이지 않은 메모리 페이지를 스왑 공간으로 복사한다.(swap out)
    - 어떤 프로그램이 스왑 아웃된 메모리 페이지에 접근하려할 경우, 커널은 다른 메모리 페이지를 스왑 아웃하여 공간을 확보한 뒤, 요청받은 페이지를 스왑 공간으로부터 물리적 메모리로 스왑 인 해야한다.
3. 주변장치 관리
    - 입/출력 장치 스케줄링 및 전반적인 관리 담당
4. 파일 시스템 관리
    - 디렉토리의 생성과 삭제, 변경, 유지 등의 관리 담당
    > 디렉토리 Directory : 파일들과 하위 디렉토리들을 포함하는 묶음

<br/>

> 커널이 이러한 기능을 하는 이유는 사용자가 물리적인 하드웨어에 접근하고 사용할 수 있도록 매개하기 위해서이다.<br/>
> 즉, <span style="color:#9fb584">**커널은 사용자가 하드웨어에 접근하고 통신하기 위한 중간 다리 역할을 수행**</span>한다.<br/>
> 사용자 -> Shell -> Kernel -> 컴퓨터 자원 접근

<br/>

### 2. 쉘 Shell

- 사용자가 입력시킨 <span style="color:#9fb584">**명령어 라인을 읽어 필요한 시스템 기능을 실행시키는 명령어 해석기**</span>로, 즉 <span style="color:#9fb584">**커널과 사용자간 다리 역할을 수행**</span>한다.
- <span style="color:#9fb584">**시스템과 사용자 간 인터페이스를 제공**</span>한다.
- 여러 내장 명령어를 가지고 있다.
- CLI(Command Line Interface) : 사용자가 직접 명령어를 입력하여 컴퓨터에 명령을 내리는 방식
- GUI(Graphical user Interface) : 마우스로 화면을 클릭하여 그래픽 위주로 컴퓨터를 제어하는 방식

## iOS 운영체제 구조

---

- iOS 운영체제의 구조는 <span style="color:#9fb584">**계층기반으로, 의사소통이 직접적으로 이뤄지지 않는다.**</span>
- 애플리케이션 레이어와 하드웨어 레이어 사이에 있는 레이어가 통신에 도움을 준다.
- 하위 레벨은 모든 애플리케이션이 의존하는 기본 서비스를 제공한다.
- 상위 레벨은 그래픽 및 인터페이스 관련 서비스를 제공한다.
- 대부분 시스템 인터페이스에는 프레임 워크라는 특수 패키지가 함께 제공된다.
  - 프레임워크는 .a 파일, 헤더 파일, 이미지와 같은 동적 공유 라이브러리를 저장하는 디렉토리이다.
  - 각 계층에는 개발자에게 도움을 주는 프레임워크 세트가 있다.

> iOS는 애플의 모바일 운영체제로, 애플에서 개발한 오픈소스 Unix 운영체제인 Darwin 기반이다.


![architecture_ios](/assets/img/architecture_ios.png)

### CORE OS 

- <span style="color:#9fb584">**모든 iOS 기능의 기초를 형성하는 하위 수준 기능이 제공**</span>된다. 
- 메모리 관리, 파일 시스템 처리 및 스레드와 같은 기본 운영 체제 표준 기능 외에도 낮은 수준의 네트워킹, 외부 액세서리에 대한 액세스를 포함한 다양한 서비스를 제공한다.
- 64비트를 지원해 애플리케이션 성능을 가속화한다.

<details>
<summary>프레임워크</summary> 


1. 핵심 블루투스 프레임워크 (Core Bluetooth Framework)<br/> 
    - BR/EDR 및 저에너지 블루투스 장치와 상호 작용한다.<br/> <br/> 
2. 외부 액세서리 프레임워크 (External Accessories Framework)<br/> 
    - 블루투스 또는 Apple Lightning 커넥터를 통해 연결된 장치 액세서리와 통신한다.<br/> <br/> 
3. 가속화 프레임워크 (Accelerate Framwork)<br/> 
    - 고성능 및 낮은 에너지 소비에 최적화된 대규모 수학적 계산 및 이미지 계산을 수행한다.<br/> <br/> 
4. 보안 서비스 프레임워크 (Security Services Framework)<br/> 
    - 앱과 앱이 유지 관리하는 데이터에 대한 액세스를 제어한다.<br/> <br/> 
5. 로컬 인증 프레임워크 (Local Authorization Framework)<br/> 
    - 생체 인식 또는 이미 익숙한 암호를 사용해 사용자를 인증한다.<br/> 
</details>


<br/> 

### CORE SERVICES

- <span style="color:#9fb584">**Core OS 계층에서 제공하는 서비스는 Core Services 계층에 의해 추상화**</span>된다.
- <span style="color:#9fb584">**모든 앱에서 사용할 수 있는 기본 서비스를 제공**</span>한다.
- 다른 계층과 마찬가지로 Core Services 계층은 일련의 프레임워크를 제공한다.

<details>
<summary>프레임워크</summary> 


1. 계정 프레임워크: 사용자가 로그인 정보를 입력할 필요 없이 앱에서 직접 외부 계정에 액세스하고 관리할 수 있다.<br/> <br/> 
2. 주소록 프레임워크: 사용자의 연락처 정보에 액세스할 수 있다.<br/> <br/> 
3. 광고 지원 프레임워크: 앱에 광고용 식별자에 대한 액세스 권한을 부여한다.<br/> <br/> 
4. CFNetwork 프레임워크: 네트워크 구성 변경 및 네트워크 서비스에 대한 액세스를 관리한다.<br/> <br/> 
5. 핵심 데이터 프레임워크: MVC(Model View Controller) 앱의 데이터 모델을 관리하는 데 사용되는 기술이다.<br/> <br/> 
6. Core Foundation 프레임워크: 기본 데이터 관리 및 서비스 기능을 제공하는 iOS 앱용 인터페이스를 제공한다.<br/> <br/> 
7. 핵심 위치 프레임워크: 애플리케이션 위치 및 방향 데이터 제공을 지원한다.<br/> <br/> 
8. 핵심 미디어 프레임워크: 필수 데이터 유형을 사용하여 시간 기반 시청각 콘텐츠를 나타낸다.<br/> <br/> 
9. 코어 모션 프레임워크: 장치의 모든 모션 기반 데이터에 액세스하는 데 사용된다.<br/> <br/> 
10. 핵심 전화 통신 프레임워크: 이동통신사의 VoIP 지원 및 고유 ID와 같은 사용자의 셀룰러 서비스 공급자에 대한 액세스 세부 정보를 제공한다.<br/> <br/> 
11. EventKit 프레임워크: 사용자가 앱 에서 캘린더 항목을 생성, 검색 및 편집할 수 있도록 캘린더 및 미리 알림 데이터에 대한 액세스를 제공한다.<br/> <br/> 
12. 기초 프레임워크 : 데이터 저장 및 지속성, 텍스트 처리, 날짜 및 시간 계산, 정렬 및 필터링, 네트워킹을 포함하여 앱 및 프레임워크에 대한 기본 기능 계층을 제공한다. Foundation에서 정의한 클래스, 프로토콜 및 데이터 유형은 macOS, iOS, watchOS 및 tvOS SDK 전체에서 사용된다.<br/> <br/> 
13. 모바일 핵심 서비스 프레임워크: 실행 및 ID 서비스를 포함한 중요한 운영 체제 기능에 대한 액세스 및 제어를 제공한다.<br/> <br/> 
14. NewsstandKit 프레임워크: Newsstand 애플리케이션의 클라이언트 측 개발에 도움이 된다. 뉴스스탠드를 통해 사용자는 모바일 기기에 최적화된 신문과 잡지를 검색할 수 있다.<br/> <br/> 
15. PassKit 프레임워크: Wallet 앱용 패스를 생성 및 배포하고 앱에서 Apple Pay와 같은 결제를 허용한다.<br/> <br/> 
16. Quick Look 프레임워크: 앱 내에서 사용할 파일 미리보기를 생성하거나 미리보기를 쉽게 편집한다.<br/> <br/> 
17. 소셜 프레임워크: 공통 시스템 인터페이스를 활용하고 지원되는 소셜 네트워킹 사이트에 자료를 게시한다.<br/> <br/> 
18. StoreKit 프레임워크: 앱 내 구매 및 App Store와의 상호 작용을 지원한다.<br/> <br/> 
19. 시스템 구성 프레임워크: 애플리케이션이 장치의 네트워크 구성 옵션에 액세스하고 Wi-Fi 또는 셀룰러 연결이 활성화되어 있는지와 같은 장치의 연결 가능성을 확인할 수 있다.

</details>

### MEDIA LAYER

- <span style="color:#9fb584">**멀티미디어 서비스를 사용**</span>할 수 있다.
- <span style="color:#9fb584">**시스템의 전체 그래픽, 오디오 및 비디오 기술을 가능**</span>하게 한다.

<details>
<summary>프레임워크</summary> 

1. 자산 라이브러리 프레임워크: 사용자 자산의 미디어 라이브러리에 대한 액세스를 제공한다.<br/> <br/> 
2. AV Foundation 프레임워크: 시청각 리소스와 함께 작동하고, 카메라 설정을 관리하고, 오디오를 편집하고, 시스템 오디오 상호 작용을 구축한다.<br/> <br/> 
3. 핵심 오디오 프레임워크: 장치의 오디오 하드웨어와의 인터페이스를 제공한다.<br/> <br/> 
4. 핵심 그래픽 프레임워크: iOS 앱의 기본 그리기 엔진이며 사용자 정의 2D 벡터 및 이미지 기반 렌더링을 지원한다.<br/> <br/> 
5. 핵심 이미지 프레임워크: 비디오 및 움직이지 않는 사진 제어를 위한 고급 비파괴 지원을 제공한다.<br/> <br/> 
6. 핵심 MIDI 프레임워크: 하드웨어 키보드 및 신디사이저를 포함한 MIDI(악기 디지털 인터페이스) 장치와 통신하기 위한 API를 제공한다.<br/> <br/> 
7. 핵심 텍스트 프레임워크: 텍스트 레이아웃 및 글꼴 처리를 위한 낮은 수준의 프로그래밍 인터페이스를 제공한다. Core Text 레이아웃 엔진은 고성능, 사용 편의성 및 Core Foundation 과의 긴밀한 통합을 위해 설계되었다. 텍스트 레이아웃 API는 문자-글리프 변환, 합자, 커닝 등을 포함한 고품질 조판을 제공한다.<br/> <br/> 
8. 핵심 비디오 프레임워크: 프레임별 편집을 포함하여 디지털 비디오를 처리하기 위해 Metal 및 OpenGL을 모두 지원하는 파이프라인 기반 API를 활용한다.<br/> <br/> 
9. 이미지 I/O 프레임워크: 이미지의 메타데이터에 대한 액세스를 제공하고 대부분의 이미지 파일 형식을 읽고 쓴다.<br/> <br/> 
10. GLKit 프레임워크: 하드웨어 가속 API를 사용하여 고급 2D 및 3D 렌더링을 관리한다.<br/> <br/> 
11. 미디어 플레이어 프레임워크: 앱 내에서 노래, 오디오 팟캐스트, 오디오북 및 기타 미디어를 찾아 재생한다.<br/> <br/> 
12. OpenAL 프레임워크: 업계 표준 오디오 전달 기술이다.<br/> <br/> 
13. OpenGL ES 프레임워크: 하드웨어 가속 인터페이스를 통해 강력한 2D 및 3D 렌더링을 제어한다.<br/> <br/> 
14. Quartz Core 프레임워크: 사용자가 슬라이드쇼와 Core Image 필터를 사용하여 사진을 보고, 변경하고, 저장할 수 있다.

</details>

### COCOA TOUCH (Application Layer)

- <span style="color:#9fb584">**아이폰 및 기타 iOS 장치 프로그래밍을 위한 다양한 라이브러리에 액세스할 수 있도록 하는 추상화 레이어를 제공**</span>한다.
- Mac OS XCocoa API를 사용하여 생성된 Objective-C 프레임워크의 필수 그룹이 Cocoa Touch 계층에 포함되어 있다.
- 알림, 멀리태스킹, 터치별 입력, 모든 고급 시스템 서비스 및 기타 중요한 기술을 지원한다.
- 앱에 대한 <span style="color:#9fb584">**기본적인 인프라 지원도 제공**</span>한다.

<details>
<summary>프레임워크</summary> 

1. 주소록 UI 프레임워크: 사용자의 연락처를 획득하여 이를 그래픽 인터페이스에 표시한다.<br/> <br/> 
2. 이벤트 키트 UI 프레임워크: 뷰 컨트롤러를 사용하여 이벤트를 표시하고 변경하는 일반적인 시스템 인터페이스를 설명한다.<br/> <br/> 
3. Game Kit 프레임워크: 사용자가 Game Center를 통해 온라인으로 게임 관련 데이터를 공유할 수 있다.<br/> <br/> 
4. iAd 프레임워크: 앱에서 배너 광고를 표시할 수 있다.<br/> <br/> 
5. 지도 키트 프레임워크: 앱의 사용자 인터페이스에 포함될 수 있는 스크롤 가능한 지도를 제공한다.<br/> <br/> 
6. 메시지 UI 프레임워크: 사용자가 앱을 종료하지 않고도 메시지를 업데이트하고 보낼 수 있도록 이메일 및 문자 메시지 작성 인터페이스를 구축한다.<br/> <br/> 
7. UI 키트 프레임워크: iOS용 그래픽 이벤트 기반 앱을 개발하기 위한 중요한 기반을 제공한다. UI Kit 프레임워크의 가장 중요한 기능 중 일부는 다음과 같다.<br/>
    - 멀티태스킹을 지원<br/>
    - 기본 앱 관리 및 인프라<br/>
    - 사용자 인터페이스 관리<br/>
    - 터치 및 모션 이벤트 지원<br/>
    - 잘라내기, 복사, 붙여넣기를 지원

</details>
